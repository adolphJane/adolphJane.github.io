CMakeLists语法介绍

PROJECT关键字

可以用来指定工程的名字和支持的语言，默认支持所有语言

  • PROJECT (HELLO) 指定了工程的名字，并且支持所有语言—建议

  • PROJECT (HELLO CXX) 指定了工程的名字，并且支持语言是C++

  • PROJECT (HELLO C CXX) 指定了工程的名字，并且支持语言是C和C++

该指定隐式定义了两个CMAKE的变量

  • _BINARY_DIR，本例中是 HELLO_BINARY_DIR
  • _SOURCE_DIR，本例中是 HELLO_SOURCE_DIR

MESSAGE关键字就可以直接使用者两个变量，当前都指向当前的工作目录，后面会讲外部编译

SET关键字

用来指定变量，Cmake中的所有变量都是String类型，可以使用set和unset命令来添加或移除一个变量

SET(SRC_LIST main.cpp) SRC_LIST变量就包含了main.cpp

也可以 SET(SRC_LIST main.cpp t1.cpp t2.cpp)

示例：

set(var 666)

message("var = ${var}")

常用变量

  • CMAKE_SOURCE_DIR 与 PROJECT_SOURCE_DIR、_SOURCE_DIR 等价: 这三个变量指代的内容是一致的，不论采用何种编译方式，都是工程顶层目录(也就是顶层 CMakeLists.txt 源码所在目录)。
  • CMAKE_BINARY_DIR、PROJECT_BINARY_DIR、_BINARY_DIR 等价: 这三个变量指代的内容是一致的。如果是内部编译，指得就是工程顶层目录(也就是顶层 CMakeLists.txt 源码的BINARY_DIR)，如果是外部编译，指的是工程编译发生的目录。
  • CMAKE_CURRENT_SOURCE_DIR 当前处理的CMakeLists.txt元am所在的路径
  • CMAKE_CURRENT_BINARY_DIR 如果是内部编译，它跟CMAKE_CURRENT_SOURCE_DIR 一致，如果是外部编译，他指的是target 编译目录。使用我们上面提到的ADD_SUBDIRECTORY(src bin) 可以更改这个变量的值。使用SET(EXECUTABLE_OUTPUT_PATH < 新路径>)并不会对这个变量造成影响，它仅仅修改了最终目标文件存放的路径。
  • CMAKE_MAJOR_VERSION cmake 的主版本号
  • CMAKE_MINOR_VERSION cmake 的次版本号
  • CMAKE_VERSION cmake 的版本号（主+次+修订）
  • PROJECT_VERSION_MAJOR 工程的主版本号
  • PROJECT_VERSION_MINOR 工程的次版本号
  • PROJECT_VERSION 工程的版本号
  • CMAKE_PROJECT_NAME 工程的名字
  • PROJECT_NAME 工程名，与 CMAKE_PROJECT_NAME 等价

IF关键字

基本语法

# 条件控制语法如下，满足条件的分支将会被执行
if(<condition>)
  <commands> 
elseif(<condition>)
  <commands> 
else()
  <commands>
endif()

基础条件表达式

基础条件表达式有三种：常量、变量、字符串

常量
if(<constant>)

<constant>表示一个常量，其取值与对应逻辑值如下（不区分大小写）：

逻辑值 常量值
true 1、ON、YES、TRUE、Y、非0数字
false 0、OFF、NO、FALSE、N、IGNORE、NOTFOUNT、空字符串、-NOTFOUND结尾字符串

如果常量取值不是以上内容，则按照变量/字符串条件处理。

变量
if(<variable>)

<variable>表示一个变量，变量取值不是上述定义的false常量时其逻辑值为true，否则逻辑值为false。

注

  1. 宏参数不是变量，环境变量不可作为条件。
  2. 使用变量时，不要用${}表示变量，直接使用变量名即可。
字符串
if(<string>)

<string>表示带引号的字符串。以下情况其逻辑值为true:

  1. 字符串内容为上述定义的true常量
  2. 策略 CMP0054 未设置为 NEW，并且字符串的值恰好是受 CMP0054 行为影响的变量名称。

While关键字

set(a "")
while(NOT a STREQUAL "xxx")
    set(a "${a}x")
    message(">>>>> ${a}")
endwhile()

Foreach关键字

基本语法

foreach(<loop_var> <items>)
    <commands>
endforeach()
  • loop_var: 用于存放每一轮循环时列表中该项的值
  • items: 列表的项
  • commands: 每一轮循环执行的命令
  • endforeach(): 是foreach循环的结束指令，为了兼容可以把loop_var作为参数传入

示例

foreach(item 1 2 3)
    message("item = ${item}")
endforeach(item)

四种使用方式

  • foreach(<loop_var> RANGE <stop>): 循环值从0到指定的stop,包含stop,stop不能为负值
foreach(item RANGE 3)
    message("item = ${item}")
endforeach()

输出

item = 0
item = 1
item = 2
item = 3
  • foreach(<loop_var> RANGE <start> <stop> [<step>]): 循环从start到的stop的值,步长为step,step默认为1,三个参数都不能为负数,而且stop不能比start小
foreach(item RANGE 0 10 2)
    message("item = ${item}")
endforeach()

输出

item = 0
item = 2
item = 4
item = 6
item = 8
item = 10
  • foreach(<loop_var> IN [LISTS[<lists>]] [ITEMS[<items>]]): 循环列表lists中的项，或直接指定项items
set(W a b c)
foreach(item IN LISTS W)
    message("item = ${item}")
endforeach()

输出

item = a
item = b
item = c

set(A 1;2)
set(B 3 4 5)
set(C "6 7")
set(D 8;9 10)
set(E "")
foreach(item IN LISTS A B C D)
    message("item = ${item}")
endforeach()

输出

item = 1
item = 2
item = 3
item = 4
item = 5
item = 6 7
item = 8
item = 9
item = 10
  • foreach(<loop_var> ... IN ZIP_LISTS <lists>): 这种用法至少要3.17版本, 是组合多个列表一起循环，如果指定的多个列表项数不相等，项数少的，在loop_var为空值
set(A 1 2)
set(B 3 4 5)
foreach(pair IN ZIP_LISTS A B)
    message("pair0 = ${pair_0}, pair1 = ${pair_1}")
endforeach()

输出

pair0 = 1, pair1 = 3
pair0 = 2, pair1 = 4
pair0 = , pair1 = 5

set(A 1 2)
set(B 3 4 5)
foreach(a b IN ZIP_LISTS A B)
    message("a = ${a},b = ${b}")
endforeach()

输出

a = 1, b = 3
a = 2, b = 4
a = , b = 5

Function关键字

function(<name> [<arg1>])
    <command>
endfunction()
  • function指令 指定函数名称
  • endfunction指令 用于结束函数的定义
  • function指令中同样的函数名称作为参数

函数的调用

function(func)
    <command>
endfunction()

调用处

func()
Func()
不区分大小写

变量的作用域

  1. 在函数中调用set(var xxx)，则var在函数的作用域内的值为xxx，如果var在函数的外层作用域有定义，值为yyy，则出了函数作用域，var的值仍然为yyy，并不会变为xxx
  2. 在函数中调用set(var xxx PARENT_SCOPE)，在函数的外层作用域设置var的值为yyy，则出了函数作用域，var的值仍然为yyy，并不会变为xxx，调用函数之后，var的值就变为xxx

函数的入参

  1. ARGC 参数的个数
  2. ARGV0、ARGV1、ARGV2...ARGV#: 第#个参数的值
  3. ARGV 所有传入的参数列表
  4. ARGN 如果在定义函数时，显示的指定了两个参数，而在调用时，传入了五个参数，则后面的三个参数就存在这个变量里

MESSAGE关键字

向终端输出用户自定义的信息

主要包含三种信息：

  • SEND_ERROR，产生错误，生成过程被跳过。
  • SATUS，输出前缀为—的信息。
  • FATAL_ERROR，立即终止所有 cmake 过程.

这些信息前面会多上一个 —-

日志查看

  1. 在.cxx->cmake->debug->${ABI}->cmake_server_log.txt中查看
  2. 想即时更新日志，使用 Build -> Refresh Link C++ Projects
  3. 使用Sync Now也可以在Build中看到输出日志

ADD_EXECUTABLE关键字

生成可执行文件

ADD_EXECUTABLE(hello ${SRC_LIST}) 生成的可执行文件名是hello，源文件读取变量SRC_LIST中的内容
也可以直接写 ADD_EXECUTABLE(hello main.cpp)

也就相当于是生成的文件名是读取了我们写在set里面的文件名

上述例子可以简化的写成

PROJECT(HELLO) 
ADD_EXECUTABLE(hello main.cpp)

注意：工程名的 HELLO 和生成的可执行文件 hello 是没有任何关系的

CMakeLists常用命令

aux_source_directory(. DIR_SRCS)

查找当前目录所有源文件，并将源文件名称列表保存到DIR_SRCS

不能查找子目录

第一个参数为源文件目录，后一个参数保存源 文件名称

add_library

add_library(<name> 类型 是否构建所有 库文件)
#注意：中间的空格

添加一个库

  1. 添加库文件，名为
  2. 指定库类型，
    1. STATIC 静态库
    2. SHAERD 动态库
    3. MODULE 在使用dydl的系统有效，若不支持dydl等同于SHARED
  3. EXCLUDE_FROM_ALL 表示该库不会被默认构建
  4. source... 用来指定库的源文件

导入预编译库

  1. 添加一个已存在的预编译库，名为
  2. 一般配合set_target_properties使用
add_library(<name> 库类型 IMPORTED)
#库类型有：SHARED STATIC MODULE UNKUOWN

#比如
add_library(test SHARED IMPORTED)
#表示以导入的方式添加一个库。
 
set_target_properties(
    test #指名目标库名
    PROPERTIES IMPORTED_LOCATION #指明要设置的参数库路径
    /${ANDROID_ABI}/libtest.so
    #导入库的路径 
)

set

#设置可执行文件的输出路径(EXCUTABLE_OUTPUT_PATH是全局变量)
set(EXCUTABLE_OUTPUT_PATH [output_path])

#设置库文件的输出路径(LIBRARY_OUTPUT_PATH 是全局变量)
set(LIBRARY_OUTPUT_PATH [output_path])

#设置c++编译参数(CMAKE_CXX_FLAGS是全局变量)
set(CMAKE_CXX_FLAGS "-Wall std=c++11")

#设置源文件集合(SOURCE_FILES是本地变量即自定义变量)
set(SOURCE_FILES main.cpp test.cpp ... )

include_directories

设置头文件目录，相当于g++选项中的-I参数

#可以用相对或绝对路径，也可以用自定义的变量值
include_directories(./include ${MY_INCLUDE})

add_executable

#添加可执行文件 
add_executable(<name> ${SRC_LIST})

target_link_libraries

target_link_libraries(<name> lib1 lib2 lib3)

#如果出现相互依赖的静态库，CMake会允许依赖中包含循环依赖，如：
add_library(A STATIC a.c)
add_library(B STATIC b.c)
target_link_libraries(A B)
target_link_libraries(B A)
add_executable(main main.c)
target_link_libraries(main A)
#将若干库链接到目标库文件
#链接的顺序应当符合gcc链接顺序规则，被链接的库放在依赖它的库的后面，即如果上面的命令中，lib1依赖于lib2，lib2依赖于lib3，则在上面命令中必须严格按照lib1 lib2 lib3的顺序排序，否则会报错。

add_definitions

add_definitions(-DFOO -DDEBUG ...)
#为当前路径以及子目录的源文件加入由-D引入的define flag，用于添加编译参数。

add_subdirectory

添加包含CMakeLists.txt和源码的子项目目录

#sub_dir指定包含CMakeLists.txt和源码文件的子目录位置

#binary_dir是输出路径，一般可以不指定
add_subdirecroty(sub_dir [binary_dir])

示例

add_subdirecroty(${CMAKE_SOURCE_DIR}/../[项目名])

file

#将message写入到filename文件中，会覆盖文件原有内容
file(WRITE filename "message")

#将message写入filename文件中，会追加在文件末尾
file(APPEND filename "message")

#将filename文件中读取内容并存储到var变量中，如果制定了numBytes和offset
#则从offset处开始最多读numBytes个字节，另外如果指定了HEX参数，则内容会以十六进制形式存储在var变量中
file(READ filename var [numBytes] [offset] [HEX])

#重命名文件
file(RENAME <oldname> <newname>)

#删除文件
file(REMOVE [file file1 ....])

#递归的执行删除文件命令，等于rm -f
file(REMOVE_RECURSE [file file1...])

#根据指定的url下载文件
#timeout超时时间，下载的状态会保存到status中，下载日志会被保存到log，sum指定所有下载文件预期的MD5值，如果指定会自动进行比对，如果不一致，则返回一个错误。
#SHOW_PROGRESS进度信息会以状态信息的形式被打印出来
file(DOWNLOAD url file [timeout] [status] [log] [sum] [show_progress])

#创建目录
file(MAKE_DIRECTORY [dir dir1 ...])

#它会把path转换为以unix的/开头的cmake风格路径，保存在result中
file(TO_CMAKE_PATH path result)

#它会吧cmake风格的路径转换为本地路径风格:window下用"\"，而unix下用"/"
file(TO_NATIVE_PATH path result)

#将会为所有匹配查询表达式的文件生成一个文件list，并将改list存储进变量variable里，如果一个表达式指定了path，返回的结果里将会是相对于给定路径的相对路径，查询表达式例子*.cxx,*.vt
#NOTE:按照官方文档的说法，不建议使用fileGLOB指令来收集工程的源文件
file(GLOB variable [path] [globbing expressions] ...)
#注意：直接添加新文件会报错，cmake不会重新生成，需要改动下cmake文件（例如添加一个空格）

set_directory_properties

  • 设置某个路径的一种属性
  • prop1，prop2代表属性
  • INCLUDE_DIRECTORIES
  • LINK_DIRECTORIES
  • INCLUDE_REGULAR_EXPRESSION
  • ADDITIONAL_MAKE_CLEAN_FILES
set_directory_properties(PROPERTIES prop1 value1 prop2 value)

set_property

  • 在给定的作用域内设置一个命名的属性
  • PROPERTY参数是必须的
  • 第一个参数决定了属性可以影响的作用与
  • GLOBAL：全局作用域
  • DIRECTORY：当前路径，可以用[dir]制定路径
  • TARGET：目标作用域，可以时0个或多个已有的目标
  • SOURCE：源文件作用域，可以是0个或多个源文件
  • TEST：测试作用域，可以是0个或多个已有的测试
  • CACHE：必须制定0个或多个cache中已有的条目
set_property(<
            GLOBAL |
            DIRECTORY [dir] |
            TARGET [target ...] |
            SOURCE [src ...] |
            TEST [test1 ...] |
            CACHE [entry1 ...] |
            [APPEND]
            PROPERTY <name> [value ...]>)
