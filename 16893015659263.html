MiniConda的安装与使用

2023/07/14 posted in  Python

miniconda与anaconda：

两者都是包管理工具，可以用来构建多个虚拟环境，为所需的项目单独构建环境，不至于造成多个项目间的环境冲突等。anaconda是在其中已经安装了部分模块，可以让使用者在使用一些常用的模块时，不用再次下载了，而且有着图形化界面；与之相比，miniconda则是简化版的conda管理工具，功能相同，只是没有提前安装那些模块，使用者可以根据自己的需要下载相应的模块，也没有图形化界面，但是下载安装的效率会超过anaconda，占用内存也比较小。

miniconda的安装：

下载安装

从miniconda官网下载相应的程序包，由于下载链接的是国外的网站，所有速度可能会比较慢，也可以用国内的清华源镜像来下载，然后直接双击程序安装即可。

这里选择第一个就可以了

然后更改安装目录，建议不要安装在C盘

之后全部勾选

最后直接Finish即可。

安装完成后，点击左下角开始图标，选择图中的箭头指向的conda命令窗口

换源

由于conda的数据仓库都是放在国外的网站，如果我们不换源的话，平时使用时的速度可能会特别慢，因此我们要把链接的仓库换成国内的。

从开始程序界面打开Anaconda Prompt面板

执行命令

conda config --set show_channel_urls yes

进入C盘->Users，找到你用户名字的文件夹，进入之后会找到.condarc这个文件

打开.condarc文件，然后删除里面的所有东西，替换成以下命令：

channels:
  - defaults
show_channel_urls: true
default_channels:
  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main
  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/r
  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/msys2
custom_channels:
  conda-forge: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud
  msys2: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud
  bioconda: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud
  menpo: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud
  pytorch: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud
  pytorch-lts: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud
  simpleitk: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud

在Anaconda Prompt面板运行以下命令，清除索引缓存，保证用的是镜像站提供的索引

conda clean -i

miniconda的相关指令

  1. 创建环境，在自定义的名字后面，还可以跟上这个环境需要的python包等
conda create -n your_name [python=3.8]
  1. 激活（切换环境），在创建了一个新的环境之后，我们需要先通过以下命令激活环境，也可以通过以下命令切换环境
conda activate your_name
  1. 退出当前环境，我们可以使用 deactivate 命令
conda deactivate
  1. 安装模块，在一个环境中，我们可以通过以下命令安装我们所需的模块
conda install module_name
  1. 查看环境，使用以下指令可以查看所有环境，*所在是我们当前所处的环境
conda info -e
或者
conda env list
  1. 删除环境，如果某一个环境我们不想要了，可以通过以下指令删除整个环境
conda remove --name myenv --all
  1. 搜索软件包
    我们可以使用 conda search 命令来搜索包含特定软件包的 Conda 仓库。例如，要搜索 pandas 软件包，运行以下命令：
conda search pandas
  1. conda中第三方库的安装与删除

使用conda install指令安装的就用conda remove package-name将包移除
使用pip install指令安装的就用pip uninstall package-name将包移除

  1. 升级软件包
    我们可以使用 conda update 命令来升级已安装的软件包。例如，要升级 pandas 到最新版本，运行以下命令：
conda update pandas
  1. 在conda中下载速度过慢，可以尝试将下载路径更换为国内的镜像源进行下载
    使用以下指令可以查看当前下载的频道，默认channels是default，这个官方下载渠道
conda config --show

使用以下指令可以更换国内镜像源地址

conda config --add channels 国内镜像源地址
 
清华镜像：
https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/msys2/
https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/conda-forge/

使用以下指令可以还原下载路径

config config --remove-key channels

问题

failed with initial frozen solve. Retrying with flexible solve.

我们用conda install xxx来安装包时，经常会遇到如下问题：

failed with initial frozen solve. Retrying with flexible solve.
Solving environment: failed with repodata from current_repodata.json, will retry with next repodata source.

这其实是conda的环境不适配，我们需要将对环境进行更新，通过如下几个步骤：

# 查看版本
conda -V
# 更新conda环境
conda update -n base conda
# 更新conda的所有包
conda update --all

做完这些之后，再用conda install xxx就不会有上面的问题了。
