ASM - MethodVisitor

2022/08/03 posted in  Gradle

通过调用ClassVisitor类的visitMethod()方法，会返回一个MethodVisitor类型的对象。

public MethodVisitor visitMethod(int access, String name, String descriptor, String signature, String[] exceptions);

MethodVisitor类

从类的结构来说，MethodVisitor类与ClassVisitor类和FieldVisitor类是非常相似性的。

class info

第一个部分，MethodVisitor类是一个abstract类。

public abstract class MethodVisitor {
}

fields

第二个部分，MethodVisitor类定义的字段有哪些。

public abstract class MethodVisitor {
    protected final int api;
    protected MethodVisitor mv;
}

constructors

第三个部分，MethodVisitor类定义的构造方法有哪些。

public abstract class MethodVisitor {
    public MethodVisitor(final int api) {
        this(api, null);
    }

    public MethodVisitor(final int api, final MethodVisitor methodVisitor) {
        this.api = api;
        this.mv = methodVisitor;
    }
}

methods

第四个部分，MethodVisitor类定义的方法有哪些。在MethodVisitor类当中，定义了许多的visitXxx()方法，我们列出了其中的一些方法，内容如下：

public abstract class MethodVisitor {
    // 访问方法一个参数
    public void visitParameter(String name, int access);
    // 访问接口方法的默认值
    public AnnotationVisitor visitAnnotationDefault();
    // 访问方法的一个注解
    public AnnotationVisitor visitAnnotation(String desc, boolean visible);
    // 访问方法签名上的一个类型的注解
    public AnnotationVisitor visitTypeAnnotation(int typeRef, TypePath typePath, String descriptor, boolean visible);
    // 访问注解参数数量，就是访问方法参数有注解参数个数
    public void visitAnnotableParameterCount(int parameterCount, boolean visible);
    // 访问参数的注解，返回一个AnnotationVisitor可以访问该注解值
    public AnnotationVisitor visitParameterAnnotation(int parameter,String desc, boolean visible);
    // 访问方法属性
    public void visitAttribute(Attribute attr);
    // 开始访问方法代码，此处可以添加方法运行前拦截器
    public void visitCode(); 
    // 访问方法局部变量的当前状态以及操作栈成员信息，方法栈必须是expanded格式或者compressed格式，该方法必须在visitInsn方法前调用
    public void visitFrame(int type, int numLocal, Object[] local, int numStack, Object[] stack);
    // 访问零操作数指令。
    public void visitInsn(final int opcode);
    // 访问数值类型指令
    public void visitIntInsn(final int opcode, final int operand);
    // 访问一个局部变量指令。局部变量指令是加载或存储局部变量值的指令。
    public void visitVarInsn(final int opcode, final int var);
    // 访问类型指令，类型指令会把类的内部名称当成参数Type
    public void visitTypeInsn(final int opcode, final String type);
    // 访问字段指令。字段指令是加载或存储对象的字段值的指令。
    public void visitFieldInsn(final int opcode, final String owner, final String name, final String descriptor);
    // 访问方法指令。方法指令是调用方法的指令
    public void visitMethodInsn(final int opcode, final String owner, final String name, final String descriptor, final boolean isInterface);
    // 访问invokedynamic(动态类型)指令
    public void visitInvokeDynamicInsn(final String name, final String descriptor, final Handle bootstrapMethodHandle, final Object... bootstrapMethodArguments);
    // 访问跳转指令。跳转指令是可以跳转到另一条指令的指令
    public void visitJumpInsn(final int opcode, final Label label);
    // 访问一个标签。标签指定在它后面要访问的指令。
    public void visitLabel(final Label label);
    // 访问LDC指令。注意，在Java虚拟机的未来版本中可能会添加新的常量类型。为了容易地检测新的常量类型，这个方法的实现应该检查意外的常量类型
    public void visitLdcInsn(final Object value);
    // 访问一个IINC指令
    public void visitIincInsn(final int var, final int increment);
    // 访问TABLESWITCH(跳转)指令
    public void visitTableSwitchInsn(final int min, final int max, final Label dflt, final Label... labels);
    // 访问LOOKUPSWITCH(查询跳转)指令
    public void visitLookupSwitchInsn(final Label dflt, final int[] keys, final Label[] labels);
    // 访问多维数组指令
    public void visitMultiANewArrayInsn(final String descriptor, final int numDimensions);
    // 访问指令上的注释。这个方法只能在注释指令之后调用。对于同一条指令，它可以被调用多次。
    public AnnotationVisitor visitInsnAnnotation(int typeRef, TypePath typePath, String descriptor, boolean visible);
    // 访问一个try catch块
    public void visitTryCatchBlock(final Label start, final Label end, final Label handler, final String type);
    // 访问异常处理程序类型上的注释。这个方法必须在{@link #visitTryCatchBlock}之后使用，用于注释异常处理程序。对于同一个异常处理程序，可以多次调用它。
    public AnnotationVisitor visitTryCatchAnnotation(int typeRef, TypePath typePath, String descriptor, boolean visible);
    // 访问局部变量声明
    public void visitLocalVariable(String name, String descriptor, String signature, Label start, Label end, int index);
    // 访问局部变量类型上的注释
    public AnnotationVisitor visitLocalVariableAnnotation(int typeRef, TypePath typePath, Label[] start, Label[] end, int[] index, String descriptor, boolean visible);
    // 访问行号声明
    public void visitLineNumber(int line, Label start);
    // 访问该方法的最大堆栈大小和最大局部变量数量
    public void visitMaxs(int maxStack, int maxLocals);
    // 访问方法的末尾。
    // 该方法是最后一个被调用的方法，用于通知访问器该方法的所有注释和属性都已被访问
    public void visitEnd(); 
}

使用方法

MethodVisitor 类的方法必须按以下顺序调用（在这个类的 Javadoc 中规定）：

visitAnnotationDefault?
( visitAnnotation | visitParameterAnnotation | visitAttribute )* ---对于非抽象方法，如果存在注释和属性的话，必须首先访问
(visitCode
( visitTryCatchBlock | visitLabel | visitFrame | visitXxxInsn |visitLocalVariable | visitLineNumber )*
visitMaxs ---visitCode与visitMaxs成对出现，出现?次。
)?
visitEnd

visistCode

ASM开始扫描这个方法

visitVarInsn(int opcode, int var)

访问局部变量指令。 局部变量指令是加载loads或存储stores局部变量值的指令。

  • opcode: 要访问的局部变量指令的操作码。 该操作码是ILOAD, LLOAD, FLOAD, DLOAD, ALOAD, ISTORE, LSTORE, FSTORE, DSTORE, ASTORE or RET.
  • var: 要访问的指令的操作数。该操作数是局部变量的索引。

需要注意的是，没有必要为了开始访问另外一个方法，而结束当前访问的方法。
实际上，MethodVisitor实例间是完全独立的，可以用任何顺序调用（但必须在cv.visitEnd()调用之前使用）

visitInsn(int opcode)

无参数的字节码指令
当opcode为IRETURN..RETURN表示这里是方法访问语句结束

visitLdcInsn(Object cst)

LDC专用指令。LDC_W，LDC2_W已被废弃

visitTypeInsn(int opcode, String type)

带有引用类型参数的字节码指令

visitMethodInsn(int opcode, String owner, String name,String desc)

访问方法的指令。 方法指令是调用方法的指令。

  • opcode: 要访问的类型指令的操作码。可以是INVOKEVIRTUAL，INVOKESPECIAL，INVOKESTATIC或INVOKEINTERFACE。
  • owner: 方法的所有者类的内部名称 (see {@link Type#getInternalName()}).
  • name: 方法名
  • descriptor: 方法签名 (see {@link Type}).
  • isInterface: 这个方法是否定义在接口中
visitFieldInsn(int opcode, String owner, String name, String desc)

操作变量

visitMax(int maxStack, int maxLocals)

该方法是visitEnd之前调用的方法，可以反复调用。用以确定类方法在执行时候的堆栈大小。

visitEnd()

表示方法输出完毕

visitCode 和 visitMaxs 方法可用于检测该方法的字节代码在一个事件序列中的开始与结束。和类的情况一样，visitEnd 方法也必须在最后调用，用于检测一个方法在一个事件序列中的结束。

调用示例

构造方法

调用构造函数时，需要调用父类的构造函数或者类内部其他构造函数。

如果没有明显指定调用父类自定义的构造方法，那么编译器会调用默认的父类构造方法super()。
但是如果要调用父类自定义的构造方法，要在子类的构造方法中明确指定。

无参构造方法

MethodVisitor mv = cw.visitMethod(ACC_PUBLIC, "<init>", "()V", null, null);
mv.visitCode();
mv.visitVarInsn(ALOAD,0); //visitVarInsn获取变量 this
mv.visitMethodInsn(INVOKESPECIAL,"java/lang/Thread","<init>","()V");//自动添加：调用父类的构造函数
mv.visitInsn(RETURN);
mv.visitMaxs(1,1);
mv.visitEnd();

调用结果

public User() {
    //如果没有显式的调用父类构造方法，会默认调用父类无参的构造函数super()
}
有参构造方法

可以使用super()或者this():因为上面的无参的构造函数中调用了super()

mv = cw.visitMethod(ACC_PUBLIC, "<init>", "(Ljava/lang/String;I)V", null, null);
mv.visitCode();
mv.visitVarInsn(ALOAD,0); // this
mv.visitMethodInsn(INVOKESPECIAL,clazzPath,"<init>","()V"); //调用自身的无参的构造函数，
mv.visitVarInsn(ALOAD,0); // this
mv.visitVarInsn(ALOAD,1); //获取入参 name
mv.visitFieldInsn(PUTFIELD,clazzPath,"name","Ljava/lang/String;"); 
mv.visitVarInsn(ALOAD,0); //
mv.visitVarInsn(ILOAD,2); //获取入参 age
mv.visitFieldInsn(PUTFIELD,clazzPath,"age","I");
mv.visitInsn(RETURN);
mv.visitMaxs(2,3);
mv.visitEnd();

调用结果

public User(String var1, int var2) {
    this(); //间接调用 无参构造函数 中的 super();
    this.name = var1;
    this.age = var2;
}
静态构造方法

即静态语句块static {}

mv = cw.visitMethod(ACC_STATIC, "<clinit>", "()V", null, null);
mv.visitCode();
mv.visitLdcInsn(1234L);
mv.visitMethodInsn(INVOKESTATIC,"java/lang/Long","valueOf","(J)Ljava/lang/Long;"); //Long.valueOf(long);自动装箱
mv.visitFieldInsn(PUTSTATIC,clazzPath,"clazzId","Ljava/lang/Long;");
mv.visitFieldInsn(GETSTATIC,"java/lang/System","out","Ljava/io/PrintStream;");
mv.visitLdcInsn("static{} invoked...");
mv.visitMethodInsn(INVOKEVIRTUAL,"java/io/PrintStream","println","(Ljava/lang/String;)V");
mv.visitInsn(RETURN);
mv.visitMaxs(2,0); //long 占用2单位slot
mv.visitEnd();

期望运行结果

static {
    clazzId = 1234L;
    System.out.println("static{} invoked...");
}

实际运行结果

public static Long clazzId = Long.valueOf(1234L);
static {
     System.out.println("static{} invoked...");
 }
