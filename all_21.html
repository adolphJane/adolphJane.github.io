Android Studio 模板之文件

文件模板是指以一个文件为基础的模板，这种形式的模板比代码片段级别的模板要复杂一些，模板中设计到 VTL(Velocity Template Language) 语法。文件级别比代码级别能够存储更多的代码内容，而且能够通过 VTL 很大程度上提高模板的灵活度，让模板不再是一个不变的文本字符串。

art 全新的ART运行环境(Android运行时库)
bionic 基础C库源代码，Android改造的C/C++库

dalvik dalvik虚拟机
developers Android开发者参考文档
development Android应用开发基础设施相关
device Android支持的各种设备及相关配置
external android使用的一些开源的模块
frameworks 应用程序框架，Android系统核心部分，由Java和C++编写
hardware 厂商硬件适配层HAL代码，硬件适配接口
libcore Android Java核心类库
libnativehelper 动态库，实现JNI库的基础
packages 应用程序包
pdk Plug Development Kit 的缩写，本地开发套件
platform_testing 平台测试
prebuilts x86和arm架构下预编译的一些资源
sdk Android的Java层sdk和模拟器
system 底层文件系统库、应用和组件
test Android Vendor测试框架
toolchain Android工具链文件
tools Android工具文件

Makefile 全局Makefile文件，用来定义编译规则

IDEA 远程一键部署Spring Boot到Docker

CentOs安装Docker

移除旧版本

yum remove docker \
                  docker-client \
                  docker-client-latest \
                  docker-common \
                  docker-latest \
                  docker-latest-logrotate \
                  docker-logrotate \
                  docker-selinux \
                  docker-engine-selinux \
                  docker-engine

安装一些必要的系统工具

yum install -y yum-utils device-mapper-persistent-data lvm2

添加软件源信息

yum-config-manager --add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

更新 yum 缓存

yum makecache fast

安装 Docker-ce

yum -y install docker-ce

如果报package docker-ce-3:19.03.11-3.el7.x86_64 requires containerd.io >= 1.2.2-3, but none of the providers can be installed错误

进入阿里云镜像地址：https://mirrors.aliyun.com/docker-ce/linux/centos/7/x86_64/edge/Packages/找到你想要的或者最新的containerd.io包，拼接在阿里云地址后面。
可以执行下方的：

yum install -y https://mirrors.aliyun.com/docker-ce/linux/centos/7/x86_64/edge/Packages/containerd.io-1.2.13-3.1.el7.x86_64.rpm

或者自拟

接着运行sudo yum install docker-ce

启动 Docker 后台服务

systemctl start docker

镜像加速

vim /etc/docker/daemon.json

在该配置文件中加入（没有该文件的话，请先建一个)

{
  "registry-mirrors": ["http://hub-mirror.c.163.com"]
}

测试能否使用，运行hello-world

docker run hello-world

配置docker远程连接端口

 vi /usr/lib/systemd/system/docker.service

找到 ExecStart， 在ExecStart=/usr/bin/dockerd之后添加 -H tcp://0.0.0.0:2375，如下图所示

为了安全，推荐使用2376端口

2375端口会被人恶意扫描入侵

TLS配置查看Centos7 Docker配置TLS认证的远程端口的证书生成教程（shell脚本一键生成）

重新加载Docker配置

systemctl daemon-reload // 1，加载docker守护线程
systemctl restart docker // 2，重启docker

查看Docker状态

systemctl status docker
